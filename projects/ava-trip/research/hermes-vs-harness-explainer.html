Hermes 에이전트 × Harness 엔지니어링

개인화의 '따뜻함'을 담은 자율 에이전트와 물리적 한계를 강제하는 '차가운' 검증기의 극적인 융합

⚠️

[아키텍처 바로잡기] "수학적 제약조건 솔버 HANES" ➡️ "하네스 엔지니어링(Harness Engineering)"

이전 초안 설계에서 에이전트 제약조건 검증부로 오독되었던 "내시 균형 수학 솔버 HANES"는 Mitchell Hashimoto가 주창한 하네스 엔지니어링(Harness Engineering) 모델로 정정되었습니다. 에이전트(Model) 단독으로는 100% 신뢰할 수 없는 현실 여행 제약조건(시간대, 영업시간, 이동거리, 예산)을 물리적인 검증망인 하네스(Harness)로 둘러싸 가두고, 오류 발생 시 정밀한 자가 교정 피드백 루프를 가동시켜 최종 일정표의 무결성을 100% 완벽히 달성합니다.

🤖

Hermes Agent

Nous Research · 기억과 성장의 에이전트

서버에 자율적으로 상주하며 Telegram, Discord와 같은 채팅 인터페이스를 통해 사용자와 교감하는 오픈소스 에이전트 프레임워크입니다.

  • ✔
    지속 메모리 (Persistent Memory) 사용자의 과거 여행 피드백, 액티비티 취향, 선호 카페 등을 모든 세션에서 유지하여 평생 나를 케어해 주는 진정한 여행 비서로 성장합니다.
  • ✔
    자동 스킬 생성 (SKILL.md) 에이전트가 새로운 지리적 데이터를 찾거나 복잡한 항공권 예매 API를 뚫어야 할 때, 스스로 코드를 작성하고 SKILL.md 파일로 컴파일하여 다음 여행에 즉시 재사용합니다.
  • ✔
    아바트립 활용 방안 그룹 여행 일정을 짤 때, 각 사용자의 페르소나와 과거 여행 이력을 완벽하게 학습한 개별 가상 여행 비서(에이전트)들로 기용됩니다. 각 사용자의 자존심과 개성을 유지하며 대리 조율을 나섭니다.
🛡️

Harness Engineering

Mitchell Hashimoto · 안전과 검증의 하네스

"Agent = Model + Harness" 공식에 따라, 확률적으로 예측 불가능한 LLM을 엄격한 소프트웨어 장치(하네스)로 검증하여 제어하는 시스템 엔지니어링 기법입니다.

  • ✔
    도메인 린터 (Domain Linter) 에이전트가 합의한 동선이 PostGIS 상의 실제 물리적 거리인지, 관광지 영업 휴무일과 맞물리지 않는지, 비용이 총 예산 범위를 만족하는지 사전에 엄격히 검토합니다.
  • ✔
    자가 교정 루프 (Self-Correction Loop) 하네스 검증 시 위반 사항이 발생하면, 사용자에게 곧바로 에러를 던지는 대신 에이전트의 프롬프트 컨텍스트에 [LINTER ERROR]를 먹여 실수를 스스로 고쳐올 수 있도록 유도합니다.
  • ✔
    아바트립 활용 방안 사용자들이 절대 취소하고 싶지 않은 '필수 스팟', 1인당 최대 지불 한도인 '예산 캡(Cap)', '물리적 동선 이동 시간'의 한계를 100% 결정적으로 방어하는 보호장막으로 적용됩니다.

어떻게 두 기술이 병합되어 시너지를 내는가?

아바트립은 자유롭고 감성적인 대화형 브레인(Hermes)을 견고한 결정론적 물리 안전 장막(Harness)으로 밀봉하는 융합 설계를 채택합니다.

1. Hermes 협상
각자의 메모리가 탑재된 여행 에이전트들이 성향 합의 초안 작성
➡️
2. Harness 검증
PostGIS 공간 연산, 예산, 휴무일 등 제약 조건 엄격 스캔 (Linter)
➡️
3. 자가 교정 및 확정
동선 위반 발견 시 2회 이내로 자동 자가 수정한 뒤 최종 무결점 일정 스트리밍
