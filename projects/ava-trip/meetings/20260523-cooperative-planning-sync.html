기획-개발 Sync 세션 가이드

아바트립 1차 기획-개발 얼라인먼트

에이전트 조율 파이프라인의 실무 구현을 위해 기획 파트와 합의해야 할 핵심 연동 요건

회의 목적

사용자의 취향 데이터를 수집하고, 이를 AI 에이전트 간의 타협 프로세스 및 물리 제약 검증기(Harness)에 동적으로 연동하기 위한 설계 요건을 조율하고 의사결정을 완료합니다.

💻 아키텍처 스펙 및 백엔드 기능 명세

시스템 구현상 기획 요건과 직접 맞물리는 3대 코어 기술 정의입니다.

👤

1. 페르소나 컨텍스트 주입

사용자가 온보딩 과정에서 선택한 취향 프로필(예산 상한, 최대 허용 이동 거리 등)을 PostgreSQL DB에서 조회하여 각 에이전트 세션의 시스템 프롬프트에 동적 삽입(RAG)합니다.

🔄

2. 비동기 병렬 처리 모델

3인의 서브 에이전트 호출을 비동기 병렬(Async Parallel) 구조로 동시 처리하여 API 지연 시간(Latency)을 최소화하고 리소스를 절감합니다.

🛡️

3. TravelHarness Linter

LLM과 분리된 순수 파이썬 노드입니다. PostGIS 공간 연산으로 실제 이동 경로를 산출하고, 사칙연산으로 예산을 체크하여 물리 제약 위반(Linter Fail) 여부를 최종 판정합니다.

📋 기획-개발 정렬을 위한 4대 핵심 의제

PRD와 유저 시나리오를 구성할 때 기술 구현 제약과 합의를 도출해야 할 세부 설계 요건입니다.

의제 01 • 성향 데이터 정량화 온보딩 데이터의 데이터베이스 매핑 규격 설계

성향 질문지의 추상적 선택지를 하네스 검증 규칙(Linter Rules)이 연산할 수 있는 명확한 정량적 수치로 가공해야 합니다.

세부 검토 사항
  • 소비 성향(가성비형/지출형)에 따른 1일 가용 예산의 구체적 구간 한도 설정 (ex. 인당 5만 원, 10만 원, 15만 원 등).
  • 이동 선호도(단거리 선호/상관없음)에 따른 하루 최대 이동 거리(km) 상한선 기준 수립 (ex. 30km, 50km 등).
의제 02 • 비동기 UX 연출 AI 협상 대기 상태에서의 로딩 연출 사양 결정

부모 에이전트가 1차 초안을 수립하고 서브 에이전트 검토를 취합하는 동안의 로딩 상태(Loading State) UX 연출 사양을 확정합니다.

세부 검토 사항
  • 에이전트들이 타협하는 시뮬레이션 로그(ex. "서핑을 추가하여 민수님의 취향 만족도가 증가했습니다")를 SSE 스트리밍으로 렌더링하는 사안 검토.
의제 03 • 예외 처리 룰 제약 조건 충돌(Deadlock) 발생 시 시스템 대응 사양 수립

에이전트 간의 조건 대립(ex. 극단적으로 낮은 예산 조건과 비싼 레포츠 필수 조건의 충돌)으로 하네스 검증이 최종 실패할 때의 예외 처리 비즈니스 룰을 합의합니다.

세부 검토 사항
  • 지속 검증 실패 시, 우선순위가 낮은 옵션을 자동 배제하는 알고리즘 작동 여부 결정.
  • 사용자 화면에 충돌 요인을 명시하고 조율을 유도하는 중재용 팝업/모달 UI 제공 사양 조율.
의제 04 • 프로토타입 범위 1차 마일스톤 MVP 구현 범위 락(Lock)

안정적인 무결성 검증과 공공 API(TourAPI) 통합을 위해 프로토타입 범위의 명확한 경계선을 설정합니다.

세부 검토 사항
  • 1차 POC 단계의 지리적 한계를 '강원도 삼척/강릉 권역'으로 국한하고, 일정 기간을 '당일 코스'로 제약하는 사안 동의 여부.

📝 의사결정 체크리스트 (Sync Board)

회의 도중 의사결정이 완료된 항목을 실시간으로 체크하며 트래킹할 수 있습니다.
