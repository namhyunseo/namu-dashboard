2026 관광데이터 활용 공모전 제출용 제안서 초안입니다.

『2026 관광데이터 활용 공모전』 — ① 웹·앱 개발 부문 제안서

1. 서비스 기획배경 및 필요성

1-1. 서비스 기획 배경

  1. 기획 배경
  • 그룹 여행의 가장 큰 장벽: '장소 선택'이 아닌 '합의 과정'.
  • 문제: 2인 이상이 함께 여행을 계획할 때, 각자의 선호·예산·일정이 달라 조율에 많은 시간과 감정적 소모가 발생.
  • 데이터 근거: 미국인 1,007명 대상 설문조사 결과, 커플 여행 갈등 원인 1위는 예산(36%), 3위는 여행 일정(28%)으로 나타남. 계획 단계의 조율 실패가 실제 갈등으로 직결됨.
  1. 기존 서비스의 한계
  • 기존 AI 여행 플래닝 서비스는 개인 맞춤형 추천에 최적화되어 있으나, 여러 사람의 의견을 동시에 반영하는 기능이 부재함.
  1. 해결이 필요한 핵심 문제 (Pain Point)
  • 의견 충돌: 구성원마다 다른 선호와 기준으로 인한 갈등.
  • 반복되는 협의: 결론 없이 이어지는 메신저 논의.
  • 결정 지연: 합의 실패로 인한 여행 계획 자체의 무산.

1-2. 서비스 필요성

  1. 여행 정보 과잉으로 인한 선택 피로 증가.
  2. 동행자 간 예산·선호도 차이로 일정 조율의 어려움.
  3. 기존 플랫폼의 예약 중심 구조로 의사결정 지원 부족.
  4. 데이터 기반 일정 조율 기능의 필요성.
  5. 갈등 감소와 여행 만족도 향상 기대.

2. 서비스 개요

2-1. 기획 서비스 소개 (한 줄 요약)

아바트립(AVA-Trip): 개인 맞춤형 AI 에이전트 간 대리 협상을 통한 갈등 없는 최적의 여행 플래닝 서비스

2-2. 기획 서비스 주요 기능

  1. 페르소나 기반 'AI 에이전트' 생성 기능
  • 사용자 데이터 수집: 성향, 선호 장소, 일정, 예산 등을 기반으로 맞춤형 AI 에이전트 생성.
  • 페르소나 프리셋: 힐링, 액티비티 등 대표적인 여행 성향 프리셋 제공.
  • 우선순위 설정: '필수(타협 불가)' 항목과 '선호(타협 가능)' 항목을 스코어링.
  1. 단계별 AI 에이전트 간 실시간 대리 협상 기능
  • 자율 협상: 에이전트 간 우선순위 연산을 통해 합의점을 도출하고 대화형 로그를 실시간 중계(게이미피케이션).
  • 중간 피드백: 협상 난항 시 사용자에게 문제점 보고 및 대안 선택 요청.
  • 사용자 개입: 실시간 조건 수정 및 특정 장소/일정 추가 가능.
  • 협상 완료: 전원 승인 시 확정, 반려 시 피드백을 반영한 2차 협상 착수.
  1. 관광 데이터 기반 최적 일정 자동 생성 기능
  • 데이터 시각화: TourAPI를 활용한 고화질 사진, 영업 시간, 리뷰 등 정보 제공.
  • 최적 경로 제안: 대중교통·도보 소요 시간 분석 및 최적 동선 제공.
  • 지도 연동: 실시간 길 찾기 및 내비게이션 기능 연동.

2-3. 서비스 차별성

  • 그룹 여행의 본질적 페인 포인트('합의 과정')에 집중.
  • 멀티-AI 에이전트 환경: 독립적인 n개의 에이전트가 다자간 소통하여 일정을 확정.
  • 감정 소모 제거: 직접적인 마찰 원인을 에이전트 대리 협상으로 대체하여 갈등 요소 원천 차단.

3. 데이터 활용 방안

3-1. 활용 예정 한국관광공사 OpenAPI

API 명칭핵심 데이터 및 활용 방식
관광지 집중률 및 방문자 추이 예측미래 시점의 혼잡도를 분석하여 쾌적도 중심의 장소 선정 근거로 활용
관광지별 연관 관광지특정 장소와 함께 방문하기 좋은 명소를 추출하여 동선 확장 및 대안 제시
국문 관광정보장소별 상세 설명, 이용 시간, 좌표 정보를 매칭해 여행 가이드 및 지도 구현
무장애 관광 정보장애인 주차장, 엘리베이터 등 편의시설 유무를 확인해 접근성 검증
고캠핑/웰니스/생태관광테마별 특화 정보를 에이전트의 전문 지식으로 활용
관광교통정보(길찾기)실시간 교통량 및 대중교통 데이터를 기반으로 경로 설정
관광 사진 정보공식 이미지를 활용해 시각적 의사결정 지원 및 계획서 시각화

3-2. 데이터 활용 방식

  1. 사용자 별 페르소나와 데이터 1:1 매칭: 에이전트가 사용자 성향에 맞는 API를 호출하여 협상 진행.
  2. 데이터 기반 에이전트 간 협상: 수집된 데이터를 비교해 제안을 검증(예: 인파가 몰리는 시간대 대신 한적한 대안 제시).
  3. 현실적 제약 반영: 이동 약자 등 특정 조건 시 무장애 API 등을 바탕으로 필수 기준 수립.
  4. 데이터 구조화: 캐싱 전략을 통한 효율적 데이터 처리 및 사용자 피드백 누적을 통한 제안 정교화.

4. 서비스 발전 방향

4-1. 데이터 축적 기반 서비스 고도화

  • 실지 선호도 지표 구축: 협상 내 수락/거절 비율을 분석하여 인지도는 낮으나 만족도가 높은 '숨겨진 명소' 발굴.
  • 비인기 지역 기피 원인 파악: 탈락 조건(접근성 등)을 데이터화하여 지역 관광 활성화 정책 근거로 제공.
  • 그룹 유형별 코스 패턴화: 누적 데이터를 활용해 유사 조건 그룹에게 '빠른 플래닝' 기능 제공.

4-2. 해외 관광객 대상 서비스 확장

  • 다국어 AI 에이전트: 영어, 중국어, 일본어 등 지원.
  • 문화적 선호 반영: 외국인 전용 페르소나 프리셋(K-culture 탐방형 등) 추가.
  • 기대효과: 서울 집중 현상 완화 및 지방 관광 활성화, 지역 소상공인 매출 기여.
