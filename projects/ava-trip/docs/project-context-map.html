🗺️ 아바트립(AVA-Trip) 프로젝트 맥락 및 절대 경로 구조 맵

For External Agents & Container Environments (e.g., Hermes Agent)

본 문서는 아바트립(AVA-Trip) 프로젝트의 핵심 개발 맥락과 디렉토리 구조를 완전한 절대 경로(Absolute Path)로 매핑한 마스터 레퍼런스입니다.

다른 작업 환경(예: Docker 컨테이너, Hermes 에이전트, 타 작업 폴더)에 본 프로젝트를 마운트하거나 이관할 때, 어떤 에이전트가 접속하더라도 이 문서를 읽고 폴더 경로를 복사하여 완벽하게 맥락을 흡수할 수 있도록 설계되었습니다.

1. 🌟 프로젝트 핵심 맥락 (Project Context)

  • 서비스명: 아바트립 (AVA-Trip)
  • 목적: 『2026 관광데이터 활용 공모전』 웹·앱 개발 부문 출품 및 실제 서비스 배포
  • 최종 데드라인: 2026년 9월 21일 (1차 심사 제출 마감)
  • 핵심 컨셉: 개인 맞춤형 여행 페르소나를 탑재한 독립적인 AI 에이전트 간의 대리 협상 및 조율을 통해, 갈등 없이 여행 일정을 자동으로 조율해 주는 PWA 모바일 플랫폼.
  • 기술 아키텍처:
  • Frontend: Next.js App Router (PWA 환경, dnd-kit 타임라인, 지도 SDK 연동)
  • Backend: FastAPI 비동기 서버 + Celery Task Queue + Redis + PostgreSQL/PostGIS (지리 공간 연산 및 캐싱)
  • AI Engine: LangGraph (순환 상태 오케스트레이션) + TravelHarness Linter (비결정성 차단 및 예산/경로 물리 제약 검증 필터)

🤖 4. 외부 에이전트(Hermes 등)를 위한 행동 수칙 및 파일 흡수 방법

본 폴더에 마운트된 다른 에이전트(AI)는 아래의 단계를 따라 프로젝트 맥락을 로드하고 태스크를 수행해야 합니다.

1단계: 마스터 컨텍스트 맵 우선 리딩

  • 에이전트는 가동 즉시 최상위의 /workspace/tourismContest/PROJECT_CONTEXT_MAP.md를 먼저 읽어 현재 어떤 서비스를 기획/개발 중인지 맥락을 완전히 습득합니다.

2단계: 필요한 작업 디렉토리의 절대 경로 복사

  • 에이전트가 만약 "마일스톤이나 요구사항 정의"를 수정하려 한다면, 이 문서의 절대 경로를 읽어 /workspace/tourismContest/01_Planning/에 있는 파일을 읽어오거나 씁니다.
  • "에이전트 조율 로직" 개발이 필요하다면 /workspace/tourismContest/03_Research/Parent_Child_Agent_Explainer.html과 Agent_Methodology_Review.md를 즉시 열어 내부 연동 규격을 모방합니다.

3단계: Inbox 현행화 수칙

  • 모든 새로운 개발 피드백이나 에이전트의 제안 사항은 임시 수집용 파일인 /workspace/tourismContest/00_Inbox/Inbox.md의 최하단에 수집 형식에 맞추어 체크리스트로 추가해야 합니다.
