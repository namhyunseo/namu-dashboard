🛠️ 아바트립 (AVA-Trip) 기술 스택 명세서 (Technical Stack Specification)

본 문서는 2026 관광데이터 활용 공모전 출품작 아바트립 (AVA-Trip)의 최종 확정된 모바일 PWA 하이브리드 앱 아키텍처 및 기술 스택을 기록합니다. 기획서의 핵심 요구사항을 현실적이고 안정적인 모바일 환경으로 구현하기 위해 최적화된 조합입니다.

💻 1. Frontend (클라이언트 & PWA 레이어)

모바일 앱 수준의 화려한 UI 인터랙션, 드래그앤드롭 타임라인, 지도 연동을 모바일 웹뷰 상에서 네이티브 수준으로 구동하는 영역입니다.

기술 명칭버전/스펙핵심 역할 및 도입 목적
Next.js (App Router)Latest (v14+)모바일 웹 성능 최적화(SSR/ISR), SEO 친화적인 페이지 구성 및 간결한 파일 기반 라우팅.
TypeScriptv5.x인터페이스와 API 스펙의 타입 안전성 확보를 통한 런타임 에러 최소화.
Vanilla CSSStandard브라우저 성능 최적화 및 Glassmorphism, 다크모드, 커스텀 물리 효과 드래그앤드롭 스타일링의 세밀한 제어.
next-pwa@ducanh2912/next-pwamanifest.json 및 Service Worker를 연동하여 모바일 홈 화면 설치(Standalone), 스플래시 화면, 푸시 알림 기반 구축.
dnd-kit@dnd-kit/core일정 조율 페이지에서 이종 일자(1일차 ➡️ 2일차) 및 시간대 간 여행지 카드를 드래그앤드롭으로 매끄럽게 재배치.
Kakao Map SDKJavaScript SDK지도 마커 렌더링, 동선 그리기 및 타임라인 마우스 오버 시 지도 자동 줌-인/포커싱 인터랙션 연동.
ZustandLatest경량화 전역 상태 관리. 사용자가 실시간으로 일정을 편집할 때 되돌리기(Undo) 및 다시실행(Redo) 기능의 이력 상태 관리.
TanStack Queryv5 (React Query)공공데이터 조회 성능 극대화를 위한 브라우저 단의 스마트 데이터 캐싱.

🧠 2. Backend (AI 에이전트 & API 레이어 - LangGraph × Harness 하이브리드)

비동기적인 LLM 협상 연산 작업을 실시간으로 백그라운드에서 처리하고, 결과를 스트리밍 형태로 사용자 모바일 화면에 전달하는 뇌 역할을 담당합니다.

기술 명칭버전/스펙핵심 역할 및 도입 목적
FastAPIPython v3.11+비동기 ASGI 기반의 초고속 API 서버 구현. Pydantic을 활용한 자동 데이터 검증 및 OpenAPI 스펙 제공.
sse-starletteLatest서버에서 진행되는 에이전트 간의 실시간 대화/협상 상태 로그(이벤트)를 모바일 브라우저로 단방향 실시간 스트리밍(SSE 구현체).
CeleryLatest수십 초 이상 소요되는 AI 협상 연산 및 하네스 검증 요청을 API 스레드와 분리하여 백그라운드 비동기 워커로 즉시 위임.
LangGraphLatest[오케스트레이션 엔진] 순환 상태 머신(StateGraph) 설계, 부모-자식 에이전트 비동기 흐름 제어, 조건부 분기(Conditional Edge) 관리 및 재귀 제한(Recursion Limit)을 통한 세션 루프 안전망 구축.
Pydantic & PostGISValidation & GIS[TravelHarness Linter] 에이전트 출력을 가로채어 지리적 거리(PostGIS), 예산 상한선, 일정 중복 여부 등의 정량 제약조건을 검사하는 결정론적 검증기.
InstructorLatest (with Pydantic)LLM 호출 시 사용자의 정성적인 성향을 완벽히 규격화된 JSON/Pydantic 스펙으로 고속 추출(Structured Output).
Google GenAI SDKGemini 1.5 Flash가볍고 빠른 LLM 연산 처리(페르소나 JSON 정량화 및 협상 대화 로그 생성).

💾 3. Database & Caching (저장소 레이어)

안정적인 데이터 관리와 공공 API 호출 오버헤드를 극복하기 위한 성능 최적화 계층입니다.

기술 명칭버전/스펙핵심 역할 및 도입 목적
PostgreSQLv15+사용자 페르소나 및 그룹 여행 일정 데이터의 안정적인 RDBMS 저장소.
PostGISExtensionPostgreSQL의 공간 정보 확장 모듈. 위경도 데이터를 활용하여 인근 관광지 검색 및 대강의 경로 간 지리적 반경 연산 성능 극대화.
RedisLatest백그라운드 작업 큐(Celery)의 메시지 브로커 및 빈번하게 참조되는 공공데이터(TourAPI 4.0)의 로컬 인메모리 캐싱 및 에이전트 상태 세션 저장소.
SQLModelLatestSQLAlchemy와 Pydantic을 결합한 ORM 라이브러리. 단일 모델 정의로 데이터베이스 테이블과 API 스펙 검증 스키마를 동시 충족.

🚀 4. 개발 및 빌드 환경 (Setup & Monorepo)

  • 레포지토리 아키텍처: FE(Next.js)와 BE(FastAPI)를 하나의 저장소 내에서 유기적으로 빌드할 수 있는 Monorepo 형태 또는 두 개의 독립된 패키지로 관리하여 개발 생산성을 보존합니다.
  • 하이브리드 확장성: Next.js 모바일 웹앱 구조를 완전하게 갖춘 상태로 유지하며, 추후 필요 시 Capacitor를 통해 Native Android/iOS 빌드로 변환할 수 있는 구조를 열어둡니다.

🔄 5. LangGraph × Harness Engineering 협업 시스템 (Integration Architecture)

본 아키텍처는 순환식 멀티 에이전트 프레임워크인 LangGraph와 물리적 제약조건을 결정론적으로 검증하는 TravelHarness를 결합하여 비동기 협상과 자가 교정 피드백 루프를 안전하게 처리합니다.

5-1. 역할 분담 명세

  • LangGraph (순환 상태 오케스트레이터):
  • 공유 상태(State) 데이터(itinerary_draft, harness_valid, harness_errors, loop_count)를 총괄 관리합니다.
  • harness_router라는 조건부 분기(Conditional Edge)를 실행하여 성공 시 END 노드로 빠지고, 실패 시 자가 교정 노드로 복귀시킵니다.
  • recursion_limit 사양을 설정하여 최대 자가 교정 시도가 초과하면 폴백 솔버(Fallback Solver)로 강제 분기하여 무한 루프를 차단합니다.
  • 부모 오케스트레이터 및 자식 서브 에이전트 (LLM 레이어):
  • 부모 에이전트(코디네이터)는 전체 일정 초안을 기획하고 수정을 총괄합니다.
  • 3인의 친구 성향을 탑재한 자식 에이전트(민수, 서연, 준호)들은 비동기 병렬(Async Parallel) 구조로 동시에 가동되어 초안 일정을 채점하고 개별 피드백을 내놓습니다.
  • TravelHarness Node (결정론적 Linter 노드):
  • LLM을 배제한 순수 Python 함수 노드입니다.
  • PostGIS 지리 쿼리로 실제 도로 최단 이동거리를 연산하고, 예산의 사칙연산을 수행하여 오류 감지 시 구체적인 에러 페이로드를 생성해 상태에 주입합니다.

5-3. 아키텍처적 이점

  1. 지연 시간(Latency) 최소화: 서브 에이전트 가동을 순차 대화 방식이 아닌 asyncio.gather 병렬 가동으로 일원화하여 성능 한계를 극복합니다.
  2. 물리적 안전성 100% 보장: LLM의 예측 불가능한 환각을 Harness가 결정론적인 공간 연산(PostGIS)으로 철저히 가로채고 방어합니다.
  3. 예외 루프 차단: LangGraph의 루프 제한 제어를 통해 에이전트 간 고집 충돌 시 발생할 수 있는 무한 루프와 API 비용 오버헤드를 사전에 예방합니다.
